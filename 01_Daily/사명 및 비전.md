### 📌 필터

- `계획일` is **Today**
    

### 📌 정렬 (위 → 아래)

### 📌 정렬

2. `마감일`
    
3. `Priority`
    

👉 주간 계획용

### 📌 정렬

👉 놓치면 안 되는 것들

---

# 1️⃣ List 뷰가 실제 작업에 더 좋은 이유

### 📌 1. 시선이 ‘업무명’에 집중됨

- Name이 크게 보임
    
- 지금 “뭘 해야 하는지” 바로 들어옴
    
- 체크리스트처럼 쓰기 좋음
    

---

### 📌 2. 속성은 “보조 정보”로만 노출 가능

- Status / 계획일 / 마감일만 작게 표시
    
- 정보 과부하 없음
    
- 판단은 빠름
    

👉 **오늘은 ‘결정’보다 ‘수행’이 목적**

---

### 📌 3. 모바일 / 태블릿에서도 압도적으로 편함

- 세로 스크롤 중심
    
- 컬럼 밀림 없음
    
- 터치 조작 스트레스 적음
    

(갤탭 S9+ 쓰시는 거 생각하면 더더욱 List 쪽이 맞습니다)

---

### 📌 4. 밀린 업무 압박이 잘 보임

- 정렬: 계획일 오래된 순
    
- 위에 쌓이는 업무 = 안 끝낸 일
    
- Table보다 심리적 신호가 강함
    

---

# 2️⃣ Table 뷰가 더 좋은 경우 (반대로)

### 📊 Table은 “관리 화면”

|상황|Table이 나은 이유|
|---|---|
|계획일 대량 수정|멀티 셀 편집|
|Priority / Type 정리|한 눈에 비교|
|Inbox 비우기|빠른 입력|
|주간 계획|구조 파악|

👉 **정리·설계·계획 세션용**

---

→ **안 보임**

이 자연스럽게 한 화면에 섞여 있습니다.
